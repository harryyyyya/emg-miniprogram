# 肌电假手管理系统 — 微信小程序技术文档

## 1. 项目概述

本小程序是面向肌电假手用户的综合管理平台，提供设备连接与控制、EMG 数据采集上传、AI 智能问答、康复社区论坛等核心功能。小程序基于微信原生框架（WXML / WXSS / JS）开发，通过蓝牙低功耗（BLE）与 ESP32-S3 硬件设备通信，后端接口基于 FastAPI（`http://127.0.0.1:8000`）。

---

## 2. 整体架构

```
微信小程序
├── pages/
│   ├── login/      登录页（微信一键 / 手机号验证码）
│   ├── index/      首页（设备状态 + 知识卡片）
│   ├── bind/       设备绑定（BLE 扫码连接）
│   ├── control/    控制页（EMG 采集 + 健康报告）
│   ├── ai/         AI 助手（LLM 对话）
│   ├── forum/      康复社区（帖子 + 评论 + 点赞）
│   └── profile/    个人中心
└── utils/
    ├── ble_manager.js   BLE 连接与数据管道管理
    ├── protocol.js      CRC32 帧协议（与后端 Python 对齐）
    └── request.js       HTTP 请求封装 + 分片上传
```

**Tab Bar 导航：** 首页 → AI助手 → 论坛 → 我的

## 3. 页面详细说明

### 3.1 登录页 `pages/login`

**功能：** 支持两种登录方式，登录成功后跳转首页。

| 登录方式 | 流程 |
|---|---|
| 微信一键登录 | 调用 `wx.login()` 获取 code → POST `/auth/wechat` → 保存 token 和 userInfo |
| 手机号验证码 | 输入手机号 → POST `/auth/sms/send` 发送验证码 → 输入验证码 → POST `/auth/sms/verify` |

**本地存储：**
- `token`：Bearer 令牌，后续所有请求自动携带
- `userInfo`：用户基本信息（id / name / phone）
- `avatarUrl`：用户头像本地路径

**降级策略：** 后端未就绪时自动触发 Mock 登录（写入 `mock_token_${timestamp}`），不影响前端开发调试。

**验证码冷却：** 发送后进入 60 秒倒计时，客户端本地实现，防止重复提交。

---

### 3.2 首页 `pages/index`

**功能：** 展示设备在线状态、Banner 轮播、知识文章入口。

**设备状态感知：**
```
onShow → 读取 boundDevice（本地存储）
       → 调用 ble.isConnected() 判断当前连接状态
       → 注册 ble.onStatus() 回调，实时感知断连/重连
```

**页面跳转：**
- `goToBind()`：跳转设备绑定页
- `goTo()`：Tab Bar 页面用 `wx.switchTab`，其他页面用 `wx.navigateTo`

---

### 3.3 设备绑定页 `pages/bind`

**功能：** 通过二维码或手动输入连接 BLE 设备，并将绑定信息同步到后端。

**绑定流程：**

```
用户扫码 / 手动输入设备名
        ↓
ble.connectDevice(name)  → 扫描 BLE 设备（超时 10s）
        ↓
连接成功 → MTU 协商（247 字节）→ 开启 Notify
        ↓
写入本地存储 boundDevice（name / hardware_id / deviceId / bindTime）
        ↓
POST /devices/bind（上报后端，失败静默处理）
        ↓
UI 跳转至"绑定成功"状态
```

**页面状态机：**

| state | 含义 |
|---|---|
| `scan` | 等待用户扫码/输入 |
| `searching` | 正在扫描连接 |
| `success` | 连接绑定成功 |
| `fail` | 连接失败，显示错误信息 |

---

### 3.4 控制页 `pages/control`

这是功能最复杂的页面，包含三个子模块。

#### 3.4.1 实时数据监控

通过 `ble.onData()` 回调接收 ESP32-S3 上报的 8 通道 500Hz EMG 数据，计算实时 RMS 强度并在环形 Canvas 图表中展示：

```javascript
RMS = sqrt( sum(v²) / (样本数 × 8通道) )
strength = min(100, round(RMS / 10))   // 映射到 0~100%
```

无真实设备时自动启动 Mock 数据流（`ble.startMock()`）。

#### 3.4.2 个性化模式 — EMG 数据采集与分片上传

**采集流程：**

```
点击"开始采集"
    ↓
发送 BLE 指令 0x0A（通知 ESP32 进入采集模式）
    ↓
按预设手势序列循环引导用户动作：
    [握拳 5s] → [张手 5s] → [握拳 5s] → [张手 5s]
    ↓
采集完毕，发送指令 0x0B 停止
    ↓
将 Int16Array 二进制数据写入本地临时文件
    路径：${USER_DATA_PATH}/emg_collect/${sessionId}.dat
    ↓
用户点击上传 → 分片上传（每片 256KB）
```

**分片上传协议：**

```
for i in range(totalChunks):
    POST /upload/chunk
    FormData: { session_id, chunk_index, file: 二进制分片 }

POST /upload/merge
Body: { session_id, total_chunks }   // 通知后端合并
```

#### 3.4.3 健康报告

- 调用 `POST /health/report/generate`，传入 `user_id`
- 后端返回诊断字段 `diagnostics`，若含"萎缩"则触发健康预警
- 后端不可用时降级展示 Mock 数据（RMS 125.3μV / 侧压力 48.7N）

### 3.5 AI 助手页 `pages/ai`

**功能：** 与后端 LLM 接口进行多轮对话，内置快捷问题入口。

**消息流：**

```
用户输入 / 点击快捷问题
    ↓
POST /ai/chat  { message, hardware_id }
    ↓
成功：渲染 AI 回复气泡
失败：触发本地 Mock 回复（按关键词匹配）
```

**快捷问题预设：**
- 我的假手昨天报错了几次？
- 最近肌电数据是否正常？
- 如何进行康复训练？
- 设备电量还剩多少？

**Mock 回复关键词映射：**

| 关键词 | 回复内容摘要 |
|---|---|
| 报错 | 过去24小时未检测到错误 |
| 肌电 / 数据 | 近7天RMS均值 125.3μV，趋势平稳 |
| 康复 / 训练 | 每日3组×15次握拳-张手交替训练 |
| 电量 | 当前约85%，可使用6-8小时 |

### 3.6 论坛页 `pages/forum`

**功能：** 康复社区，支持发帖（图文）、点赞、评论。

#### 帖子列表
- 分页加载（每页15条），支持下拉刷新、触底加载更多
- 接口：`GET /forum/posts?page=N&size=15`
- 后端不可用时展示本地 Mock 帖子数据

#### 发布帖子
- 最多上传 9 张图片（通过 `wx.chooseMedia` 选取）
- 图片逐张调用 `Forum.uploadImage()` 上传，获得 URL 后再 POST 帖子内容
- 接口：`POST /forum/posts`，Body `{ content, image_urls }`

#### 点赞
- 前端即时更新点赞状态和计数（乐观更新），再异步调用后端
- 接口：`POST /forum/posts/{id}/like`

#### 评论系统
- 点击评论图标弹出底部评论面板，支持分页加载（每页20条）
- **离线缓存策略：** 后端成功返回评论后写入 `wx.setStorageSync`；后端不可用时优先读本地缓存，本地无数据再降级到 Mock
- 发送评论：乐观追加到列表 + 写本地缓存，后端失败时弹出"已暂存"提示
- 接口：`POST /forum/posts/{id}/comments`，返回 `{ comment_id }`

### 3.7 个人中心页 `pages/profile`

**功能：** 展示用户信息与绑定设备，支持更换头像和退出登录。

| 功能 | 实现 |

## 4. 工具模块

### 4.1 BLE 管理器 `utils/ble_manager.js`

负责与 ESP32-S3（8通道 500Hz）的全部蓝牙通信。

**BLE 服务与特征 UUID：**

| 用途 | UUID |
|---|---|
| 服务 | `0000FFE0-0000-1000-8000-00805F9B34FB` |
| 数据上行（Notify） | `0000FF02-0000-1000-8000-00805F9B34FB` |
| 指令下发（Write） | `0000FF01-0000-1000-8000-00805F9B34FB` |

**连接流程：**

```
openBluetoothAdapter
    → startBluetoothDevicesDiscovery（匹配设备名/MAC，超时10s）
    → createBLEConnection
    → setBLEMTU(247)
    → getBLEDeviceServices → getBLEDeviceCharacteristics
    → notifyBLECharacteristicValueChange（开启 FF02 Notify）
    → 启动 200ms 数据处理定时器
```

**环形缓冲区：**
- 大小：4096 字节
- BLE Notify 数据写入缓冲区，200ms 定时器批量读取并解帧

**数据处理窗口（200ms）：**
```
读取缓冲区全部数据 → 逐帧识别 [0xAA...0x55]
    → protocol.unpackFrame 验证 CRC32
    → protocol.parseEMGPayload 解析为二维数组 [样本][通道]
    → 触发 onData 回调
```

**指令发送：** `sendCommand(cmdByte)` 将单字节指令打包成完整帧写入 FF01 特征。

**Mock 模式：** 无硬件时调用 `startMock()`，每 200ms 生成 100 个随机 8 通道样本（±1000 模拟肌电值），供前端开发调试。

---

### 4.2 帧协议 `utils/protocol.js`

与后端 Python `ProtocolUtils`（基于 `zlib.crc32`）完全对齐。

**帧格式：**
```
[0xAA] [payload bytes] [CRC32 4字节 little-endian] [0x55]
```

**EMG Payload 格式：**
- 每个采样点 = 8通道 × int16（little-endian） = 16 字节
- 每帧可携带多个采样点

**CRC32 算法：** 标准 IEEE 802.3 多项式（`0xEDB88320`），结果与 Python `zlib.crc32(data) & 0xFFFFFFFF` 完全一致。

| 函数 | 说明 |
|---|---|
| `packFrame(payload)` | 封包，生成完整帧 |
| `unpackFrame(buffer)` | 解包 + CRC 校验 |
| `parseEMGPayload(payload)` | 解析为 `Array<Array<number>>` |
| `buildCommand(cmdByte)` | 构造单字节指令帧 |

---

### 4.3 网络请求 `utils/request.js`

**基础请求 `request(options)`：**
- 自动拼接 BASE_URL（`http://127.0.0.1:8000`）
- 自动携带 `Authorization: Bearer {token}` 请求头
- HTTP 401 自动清除 token 并跳转登录页
- 非 2xx 响应自动弹出 Toast 错误提示

**分片上传 `uploadChunk(url, filePath, formData, onProgress)`：**
- 基于 `wx.uploadFile`，`name` 字段固定为 `file`（对应后端 FastAPI `UploadFile`）
- 支持进度回调 `onProgress(progress, bytesSent, totalBytes)`

## 5. 数据流总览

```
ESP32-S3 (BLE 500Hz)
    → BLE Notify → 环形缓冲区 → 200ms 定时解帧
    → protocol.parseEMGPayload → ble.onData()
    → control.js：实时 RMS 计算 + Canvas 渲染
                   采集模式：累积到 _collectBuffer
    → 写入本地二进制文件（Int16Array）
    → 分片上传（256KB/片） → 后端合并

微信小程序 ←→ FastAPI 后端
    登录/注册    /auth/wechat  /auth/sms/*
    设备绑定     /devices/bind
    EMG 上传     /upload/chunk  /upload/merge
    健康报告     /health/report/generate
    AI 对话      /ai/chat
    论坛         /forum/posts  /forum/posts/{id}/comments
```

## 7. 降级与 Mock 策略

| 场景 | 策略 |
|---|---|
| 后端接口不可用 | 所有 `request()` 失败进入 `.catch()`，展示 Mock 数据，不影响前端开发 |
| 无 BLE 硬件 | `ble.startMock()` 模拟 500Hz 8通道数据流 |
| 论坛评论后端不可用 | 读本地缓存 → 无缓存则展示 Mock 评论 |
| 发送评论后端失败 | 本地追加评论 + 提示"已暂存" |
| 登录后端不可用 | 写入 Mock token，直接进入首页 |
